# 3D Master Access Sheet - Full Build Spec

## Overview
A WebXR (A-Frame) 3D world for Meta Quest 3S where the user can walk through, interact with, and manage their Master Access Sheet — a 5,170-line infrastructure document with 30 sections. Think: **Walmart store layout meets Minority Report data visualization**.

## Data Source
- **Static fallback**: `data/sections.json` (already generated by `build.py`)
- **Live sync**: Fetch raw from `https://raw.githubusercontent.com/israelburns/master-access/main/MASTER_ACCESS_SHEET.txt` and parse client-side
- The JSON has 5 wings, each containing sections with full body text, stats (active/blocked/todo/done counts), subsections, and aisle codes

## World Layout

### Spawn Hub (Center)
- Circular room, ~12m diameter, dark ceiling with ambient gold lighting
- **Session Log Panel** directly in front of spawn point showing:
  - Current date/time
  - "Last visited: [date from localStorage]"
  - "Currently building: [from active session status section]"
  - "Sections accessed this session: [list]"
  - Total stats: X active, Y blocked, Z todos, W done
- **5 corridor entrances** evenly spaced around the hub perimeter (72° apart)
- Each entrance has:
  - An archway with the wing name lit in the wing's color
  - Wing description text below
  - Section count badge
  - A floor arrow/path in the wing color leading into the corridor

### Corridors (5 Wings)
Each wing is a corridor extending outward from the hub:

| Wing | Color | Sections |
|------|-------|----------|
| COMMAND CENTER | #ff4444 (red) | 6 — Laws, directives, protocols |
| OPERATIONS | #44aaff (blue) | 5 — Session status, capabilities |
| CREDENTIALS VAULT | #d4a853 (gold) | 7 — Access keys, servers, accounts |
| PROJECTS & TOOLS | #3fb950 (green) | 6 — Project links, deploy, automation |
| COMMS & LOGS | #bc8cff (purple) | 6 — Communications, session logs |

#### Corridor Structure (Walmart Aisle Logic)
- Corridor width: ~6m, walls on both sides
- Sections placed on alternating left/right walls
- **Overhead aisle signs** (like Walmart hanging signs):
  - Hang from ceiling, visible from distance
  - Show aisle code (e.g., "C1", "V3") and section title
  - Color-coded to wing color
  - Should be readable from the hub entrance (large text)
- **Floor markers** with aisle numbers
- **Shelf reference** format: `[Wing Initial][Aisle Number]` (e.g., C1 = Command Center Aisle 1, V3 = Vault Aisle 3)
- Gold strip lighting along floor edges (like runway lights)
- Each section is a **wall-mounted panel** showing:
  - Aisle code (top-left badge)
  - Section title (gold text)
  - Line count / stats preview
  - Status indicators (green dots for active, red for blocked)
  - Subsection list preview
  - "OPEN" prompt when laser-pointed at

### Section Panel Interaction (Scrolling Tablet TV)
When the user clicks/triggers on a wall panel:
1. A **floating tablet** spawns in front of the user (~1.5m away, ~2m wide × 2.5m tall)
2. The tablet displays the full section text in a monospace font
3. **Scrolling**: Shows ~30 lines at a time with:
   - Scroll UP button (top-right, clickable with laser)
   - Scroll DOWN button (bottom-right, clickable with laser)
   - Thumbstick Y-axis also scrolls when pointing at tablet
   - Page indicator: "Lines 1-30 of 245"
4. **Close button** (X) in top-right corner
5. Tablet has the wing color as border/accent
6. Semi-transparent dark background so world is visible behind it

### Grab & Carry System
- When a tablet is open, user can **grab it** with the grip button
- Grabbed tablet attaches to the controller and moves with the user
- **Release grip** = drop tablet in current world position (it stays there)
- Dropped tablets become slightly smaller (1.5m wide) and semi-transparent
- Each dropped tablet retains full scroll functionality

### Hotbar (Minecraft-style)
- Fixed HUD element at bottom of view (attached to camera)
- Shows 6 slots for carried/recently-opened tablets
- Each slot shows:
  - Aisle code
  - Abbreviated section title
  - Wing color indicator
- **Look at a hotbar slot + trigger** = re-summon that tablet in front of you
- Active slot highlighted with glow

### Beacon System
- When you pick up a tablet and walk away, a **beacon** appears at its origin wall panel:
  - Vertical pulsing light column in the wing color
  - Visible from anywhere in the corridor
  - Thin dashed line connecting beacon to the carried tablet
- Click a beacon to teleport back to that aisle location
- Beacons disappear when the tablet is returned/closed

### Tab Linking
- When holding two tablets (one in each hand, or two in hotbar):
  - A "LINK" button appears between them
  - Clicking LINK merges them into a split-screen tablet
  - Left half = first section, right half = second section
  - Both sides independently scrollable
- Linked tabs show in hotbar as a combined slot with both aisle codes

## Technical Requirements

### Stack
- **A-Frame 1.5.0** (WebXR framework)
- **aframe-extras 7.0.0** (movement-controls, hand-controls)
- **No build tools** — single HTML file or minimal file set for Quest browser compatibility
- Must work over HTTPS (Quest requirement for WebXR)

### Quest 3S Controller Mapping
- **Trigger** (index finger): Select/click/open panels
- **Grip** (side button): Grab/release tablets
- **Thumbstick Y**: Scroll content when pointing at tablet
- **Thumbstick**: Movement (via aframe-extras movement-controls)
- **A button**: Toggle hotbar visibility
- **B button**: Return to hub (teleport)

### GitHub Live Sync
- On world load, fetch master sheet from GitHub raw URL
- Parse sections client-side (same regex as build.py)
- Compare with cached version in localStorage
- If changed, rebuild affected panels with flash animation
- Show "SYNCED" or "OFFLINE (cached)" indicator in hub
- Re-sync button in hub session panel
- Poll every 5 minutes for updates (optional toggle)

### Session Tracking (localStorage)
Store and display:
```json
{
  "last_visit": "2026-03-31T09:30:00Z",
  "visit_count": 15,
  "sections_accessed": ["C1", "V3", "O2"],
  "total_time_minutes": 45,
  "session_history": [
    {"date": "2026-03-30", "sections": ["C1", "C2"], "duration": 12}
  ]
}
```

### Performance
- Lazy-load section body text (only load full text when panel is opened)
- Keep polygon count low (planes, not complex geometry)
- Use fog to hide distant corridors until approached
- Target 72fps on Quest 3S

## Visual Style
- Dark theme: background #050508, panels #0d1117
- Gold accents: #d4a853 (borders, text headers, floor lights)
- Each wing has its own accent color (see table above)
- Monospace font throughout (monoid in A-Frame)
- Subtle glow/emission on interactive elements
- Status colors: green (#3fb950) = active, red (#f85149) = blocked, orange (#f0883e) = todo
